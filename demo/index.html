Auralert

Alternativa vanilla a SweetAlert2 — demo interactiva

Toast

Tipos

Posición

Auralert.toast({
  message: 'Turno confirmado',
  type: 'success',
  duration: 3000,
  position: 'top-right'
})

Modal

const ok = await Auralert.modal({
  title: '¿Cancelar turno?',
  message: 'Esta acción no se puede deshacer',
  type: 'warning',
  confirmText: 'Sí, cancelar',
  cancelText: 'Volver',
  showCancel: true
})
if (ok) { /* confirmó */ }

Banner

Auralert.banner({
  message: 'Nueva versión disponible',
  type: 'info',
  position: 'top',
  closeable: true
})

Notify (in-app)

Auralert.notify({
  title: 'Nuevo turno',
  message: 'Juan Pérez reservó para las 15:00',
  type: 'success',
  duration: 5000
})

Tema Auralert

Auralert.setTheme('dark')  // 'light' | 'dark' | 'auto'

Integración Django

1. Script en base.html

<script src="https://TU_USUARIO.github.io/auralert/dist/auralert.min.js"></script>

2. Messages framework → toast

En tu template (ej. después del body):

{% if messages %}
<script>
document.addEventListener('DOMContentLoaded', function () {
  {% for message in messages %}
  Auralert.toast({
    message: "{{ message|escapejs }}",
    type: "{% if message.tags == 'error' %}error{% elif message.tags == 'success' %}success{% elif message.tags == 'warning' %}warning{% else %}info{% endif %}"
  });
  {% endfor %}
});
</script>
{% endif %}

En la vista:

from django.contrib import messages

def confirmar_turno(request):
    # ...
    messages.success(request, "Turno confirmado correctamente")
    return redirect("agenda")

3. AJAX / fetch

fetch('/api/reservar/', { method: 'POST', body: formData })
  .then(r => r.json())
  .then(data => {
    if (data.ok) {
      Auralert.toast({ message: data.message, type: 'success' });
    } else {
      Auralert.notify({
        title: 'Error',
        message: data.error,
        type: 'error'
      });
    }
  });
