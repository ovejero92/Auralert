Auralert

Librería JavaScript universal para alertas, toasts, modales y notificaciones. Un solo <script> — funciona en cualquier stack.

1. Instalación

Elegí una forma de cargar la librería. Solo necesitás una.

GitHub Pages (recomendado)

<script src="https://ovejero92.github.io/Auralert/dist/auralert.min.js"></script>

jsDelivr (por versión)

<script src="https://cdn.jsdelivr.net/gh/ovejero92/Auralert@1.2.0/dist/auralert.min.js"></script>

Archivo local

<script src="/ruta/a/dist/auralert.min.js"></script>

Uso mínimo

<script src="https://ovejero92.github.io/Auralert/dist/auralert.min.js"></script>
<script>
  Auralert.toast({ message: '¡Listo!', type: 'success' });
</script>

function MiComponente() {

      title: '¿Confirmar?',

}

3. Personalización

A diferencia de librerías cerradas, podés controlar clases, estilos, HTML, iconos, callbacks y botones custom en cada llamada.

// Modal con estilos y botones propios
await Auralert.modal({
  title: 'Plan Pro',
  message: '¿Querés desbloquear temas premium?',
  type: 'info',
  width: '480px',
  modalClass: 'mi-modal',
  style: { borderRadius: '20px', padding: '32px' },
  overlayStyle: { backdropFilter: 'blur(4px)' },
  confirmClass: 'aa-btn-primary',
  confirmStyle: { background: '#6366f1', borderRadius: '999px' },
  buttons: [
    { text: 'Más tarde', className: 'aa-btn-secondary', value: false },
    { text: 'Ver planes', className: 'aa-btn-primary', value: true, onClick: () => console.log('click') }
  ],
  onOpen: (el) => console.log('abierto', el),
  onClose: (v) => console.log('cerrado', v)
});

// Toast con HTML e icono custom
Auralert.toast({
  html: '<strong>Pedido #42</strong> enviado',
  icon: '📦',
  duration: 4000,
  toastClass: 'mi-toast',
  style: { background: '#1e1e2e', color: '#fff' },
  onShow: (el) => el.style.borderLeft = '4px solid #f59e0b'
});

// Defaults globales para todo el proyecto
Auralert.configure({
  toast: { position: 'bottom-right', duration: 4000 },
  modal: { confirmText: 'OK', cancelText: 'No' },
  theme: 'auto'
});

Opciones comunes (todos los componentes)

  • type — success | error | warning | info
  • className / customClass — clases extra en el contenedor
  • style — objeto de estilos inline
  • icon — emoji o texto; false para ocultar
  • html — contenido HTML (en lugar de message)
  • onShow / onClose — callbacks de ciclo de vida
  • animationDuration — duración de cierre en ms

Solo modal

  • titleHtml, overlayClass, backdrop: false
  • closeOnBackdrop, closeOnEscape, width, maxWidth
  • buttons: [{ text, className, value, onClick, style }]

5. ¿Versión Pro en el futuro? roadmap

La versión gratuita cubre todo lo esencial y es muy configurable. La idea para Auralert Pro (de pago) sería:

  • Animaciones premium (spring, morph, partículas)
  • Temas visuales: mármol, oro, glass, neón, etc. vía CSS (gradientes + texturas)
  • Packs de skins para modal/toast sin tocar tu código base
  • Licencia para uso comercial en muchos dominios
¿Se puede simular mármol u oro? Sí. Con CSS moderno (background-image, gradientes, backdrop-filter, sombras en capas) se logran efectos muy convincentes sin imágenes pesadas. En Pro podrías hacer Auralert.setSkin('marble') y aplicar variables + clases precargadas.
